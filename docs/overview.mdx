# ImageGlass_Mac — Overview

> **ImageGlass_Mac is a Mac-native image viewer.**
> It pairs a fast, ad-free viewing experience with an automation surface
> (MCP), a panel-based UI, and a scope-driven, AI-editable Local Storage
> configuration model.

---

## Most Important Rules and Goals

These are the load-bearing goals for ImageGlass_Mac. Every change in this
repository should serve one of these five goals.

### 1. MCP Support

Add a Model Context Protocol (MCP) server to ImageGlass_Mac so that
**Claude Code (and other MCP clients) can drive and configure
ImageGlass_Mac from outside the application**. The MCP server is the
integration point that lets external AI tooling read and modify the Local
Storage configuration described in Goal 4.

* MCP is how the AI talks to the viewer.
* MCP is how the AI edits scopes without the user opening the GUI.
* MCP is the contract — keep it stable, documented, and ergonomic.

### 2. Modular UI Panels (New Column)

Add a **new column** to the ImageGlass_Mac UI that hosts modular UI panels.

* The initial panel is a **directory / filename panel** that lists the images
  and files that are currently in scope.
* The same panel can be toggled into a **file-tree view** that shows the same
  files arranged in their directory hierarchy.
* The architecture is **panel-based** so additional panels can be added over
  time without disturbing the core viewer.

### 3. Scope Controls (Include / Exclude)

Give the user explicit controls to define **what is included in the active
file list and what is excluded**.

* Scopes can be expressed as directories, directory hierarchies,
  glob/extension criteria, or named rule sets.
* Scope is the single source of truth for what shows up in the panel.

### 4. Local Storage Feature

A **local-filesystem-backed configuration store**, kept as plain text files
on disk in a known directory. This is the feature name: **Local Storage**.

Each entry records:

* The **last time** the scope was evaluated / loaded.
* The list of **images (and other files)** that were resolved into the file
  list on that run.
* The list of **source directories or criteria** that drove the scope.

Runtime behavior:

* Local Storage holds a list of directories or criteria.
* It walks the matching directory hierarchies.
* It finds the files that satisfy the scope.
* That resolved list is what gets shown in the directory / filename panel
  (Goal 2).

Plain text on disk is non-negotiable — it is what makes Goal 5 possible.

### 5. MCP-Driven Editing of Local Storage

Because Local Storage state is plain text files **and** is fronted by the MCP
server, **Claude Code can**:

* Read the current scope definitions.
* Modify scope definitions.
* Swap in new directory lists.
* Change inclusion / exclusion criteria.
* Trigger re-evaluation.

…all **without the user having to edit settings inside the GUI**.

## What ImageGlass_Mac Is

ImageGlass_Mac is an open-source, ad-free photo viewer for macOS. It is
designed as a fast, modern, and highly customizable image viewer with broad
support for legacy, modern, and specialized image formats.

- An open-source image previewer / viewer focused on speed and a clean UI.
- A first-class macOS app: native window chrome, gestures, dark/light
  system theme, Apple Silicon performance.
- A platform for power users: themes, icon packs, language packs,
  command-line utilities, and integrations with third-party tools.
- Free of advertisements; funded entirely by user donations and
  sponsorships.

See `automation.mdx` for the full description of the MCP automation,
panels, scopes, and Local Storage subsystems.

## Core Selling Points

1. **90+ image formats** — WEBP, GIF, SVG, PNG, JXL, HEIC, RAW, and many
   more via the macOS ImageIO pipeline.
2. **Fast and lightweight** — minimal resource consumption; native Apple
   Silicon performance.
3. **Highly personalizable** — themes, icon packs, language packs.
4. **Clean, modern UI** — toolbar and layout are fully configurable.
5. **Third-party app integration** — launch external editors with the
   current image passed as an argument.
6. **No ads, no tracking** — fully open source, community-supported.
