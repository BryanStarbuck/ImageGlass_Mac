# App Configuration Files

ImageGlass_Mac is configured entirely through JSON files. There is no
proprietary database or binary settings store — every setting is plain
text, version-control-friendly, and easy to script.

## File Locations

There are two directories ImageGlass_Mac uses:

| Directory | Path | Notes |
| --- | --- | --- |
| **App Bundle Dir** | `/Applications/ImageGlass_Mac.app/Contents/Resources/` | Read-only; controlled by installer / admin. Contains shipped defaults. |
| **Config Dir** | `~/Library/Application Support/ImageGlass_Mac/` | Writable by the user; contains the live settings. |

## The Three Configuration Files

ImageGlass_Mac recognizes three configuration files, processed in order:

### 1. `igconfig.default.json` (App Bundle Dir)

Specifies **default settings** that apply on first launch when no other
settings exist. Ships with the app, edited by app vendors or
deployment teams to set sensible defaults for new users.

### 2. `igconfig.json` (Config Dir)

The **live user settings**. Written by the app on close. Takes precedence
over the default file. This is the file you'd back up to preserve a user's
configuration.

### 3. `igconfig.admin.json` (App Bundle Dir)

The **admin override**. Settings here override everything — defaults,
user settings, and even command-line arguments. Used to lock down
specific options in managed environments.

## Setting Priority (Lowest → Highest)

1. Built-in developer defaults (compiled into the app).
2. `igconfig.default.json` values.
3. `igconfig.json` values.
4. Command-line arguments (prefixed with `/`).
5. `igconfig.admin.json` values. **Highest priority — cannot be overridden.**

After applying all sources, ImageGlass_Mac writes the resulting merged
configuration back to `igconfig.json` for the next session.

## Command-Line Overrides

Any setting can be overridden at launch via `/Setting=Value`:

```bash
ImageGlass_Mac /ShowToolbar=false /ShowGallery=false
```

Quotes are required for values that contain spaces.

## Deployment Patterns

**Single-machine user customization**
Edit `igconfig.json` directly in `~/Library/Application Support/ImageGlass_Mac/`.
ImageGlass_Mac writes any subsequent changes here.

**Locked-down enterprise install**
1. Install ImageGlass_Mac system-wide.
2. Place `igconfig.admin.json` in the app bundle's Resources directory
   with the locked settings.
3. Optionally provide `igconfig.default.json` with non-locked defaults.

Because admin-tier values override everything, users physically cannot
change those settings through the UI without admin access to the
install folder.

**Mass deployment with custom defaults but user-editable**
Ship `igconfig.default.json` only — users will get the defaults on first
launch and any subsequent edits will persist in `igconfig.json`.
