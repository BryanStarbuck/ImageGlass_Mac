# Building Tools for ImageGlass_Mac

ImageGlass_Mac exposes a small SDK that lets external programs integrate
tightly with the viewer — receive image-loaded events, respond to user
actions, and pass the currently displayed image to your tool.

## Prerequisites

- **Xcode 16+**
- **Swift 6**
- macOS 14 (Sonoma) or later

## Step-by-Step

### 1. Create a project

Start a new Swift project in Xcode. A command-line tool, AppKit app, or
SwiftUI app all work — the SDK is UI-framework-agnostic.

### 2. Add the `ImageGlassTools` Swift package

In Xcode: **File → Add Packages…** and add the `ImageGlassTools` package,
or declare it as a dependency in `Package.swift`:

```swift
.package(url: "https://github.com/.../ImageGlassTools.git", from: "1.0.0")
```

### 3. Instantiate the tool class

```swift
let igTool = ImageGlassTool()
```

### 4. Register event handlers

The two events most tools care about:

- `toolMessageReceived` — fires when ImageGlass_Mac sends your tool a
  message (e.g., a new image was loaded).
- `toolClosingRequest` — fires when ImageGlass_Mac is shutting down or
  has asked the tool to close.

```swift
igTool.onMessage = { event in handleMessage(event) }
igTool.onClosing = { handleClosing() }
```

### 5. Implement the handlers

Messages are typed by name. The most common one:

- `IMAGE_LOADED` — a new image is now displayed in ImageGlass_Mac.
  Payload includes the full file path.

```swift
func handleMessage(_ event: ToolMessageEvent) {
    if event.name == "IMAGE_LOADED" {
        let path = event.data as? String
        // …react to the new image…
    }
}
```

### 6. Connect to ImageGlass_Mac

```swift
try await igTool.connect()
```

This opens an IPC channel back to the viewer. Once connected, events
start flowing.

## Registering the Tool in ImageGlass_Mac

There are two ways to make ImageGlass_Mac aware of your tool.

### Option A — UI

1. Open ImageGlass_Mac.
2. Go to **Settings → Tools**.
3. Click **Add…**.
4. Provide a unique tool ID, the path to your executable, optional
   command-line arguments, and an optional hotkey.

### Option B — Edit `igconfig.json`

Add an entry under the `Tools` section with:

- A unique identifier (your tool's name / ID).
- Path to the executable.
- Arguments — use the `<file>` placeholder to receive the current image
  path.
- Optional hotkey binding.
- Integration flag (whether the tool runs in-process or as an external
  child process).

## The `<file>` Placeholder

When you launch a tool from ImageGlass_Mac with arguments that include
`<file>`, ImageGlass_Mac substitutes the full path of the currently
displayed image. Example tool entry:

```json
{
  "ToolId": "my-editor",
  "Executable": "/usr/local/bin/my-editor",
  "Arguments": "--open <file>",
  "Hotkey": "⌘E"
}
```

Pressing `⌘E` while viewing `~/Pictures/sunset.jpg` would launch:

```bash
/usr/local/bin/my-editor --open "~/Pictures/sunset.jpg"
```

## Example Use Cases

- **EXIF metadata viewer** — companion metadata inspector.
- **External editors** — launch Photoshop, GIMP, Acorn, or Pixelmator
  with the current image in one keystroke.
- **Batch processors** — append the current image to a processing queue.
- **Cloud sync helpers** — upload the displayed image to a remote service.
